## [Unreleased]

### Added
- RE10K 데이터셋에도 **scene blacklist** 메커니즘 적용. `src/dataset/dataset_re10k.py`에서 `dataset_dl3dv`의 `load_blacklist`/`_resolve_blacklist_path` 헬퍼를 그대로 재사용. `DatasetRE10kCfg`에 `blacklist_path: Path | None = None` 필드 추가 (None이면 `{cfg.root}/{train|test}/blacklist.csv`). `DatasetRE10k.__init__`에서 meta.csv로 `valid_keys` 빌드 직후 blacklist set으로 key 단위 필터링하고 dropped count 출력. `build_re10k_meta` 자체는 변경 없음 — meta.csv는 cfg-agnostic하게 유지하고 필터링은 `__init__`에서만 수행해서 blacklist.csv에 key를 추가하면 meta.csv 재빌드 없이 다음 학습부터 즉시 효과. 현재 `diffusion_wrapper`의 NaN-time auto-append는 DL3DV에만 분기되어 있으므로 RE10K는 NaN 발생 → 데이터 확인 → 사용자가 수동으로 `blacklist.csv` 편집 → 학습 재시작 워크플로우.
- DL3DV scene **blacklist** mechanism. `src/dataset/dataset_dl3dv.py`에 `BLACKLIST_FILENAME = "blacklist.csv"`와 `load_blacklist` / `append_blacklist` / `_resolve_blacklist_path` 헬퍼 추가. CSV 컬럼은 `scene, reason, step, loss, detail`. `DatasetDL3DVCfg`에 `blacklist_path: Path | None = None` 필드 추가 (None이면 `{cfg.root}/train/blacklist.csv` 기본). `build_dl3dv_meta`는 scene_dir scan 직후 blacklist set으로 필터링하고 dropped count를 출력. `DatasetDL3DV.__init__`도 meta.csv 로드 직후 동일하게 chunk.name 단위로 필터 (기존 meta.csv를 재빌드하지 않고도 즉시 효과).
- `src/dataset/dataset_re10k.py`에 **camera teleport 필터** 추가. `_re10k_cameras_to_c2w(cameras)` 헬퍼가 `(N, 18)` 카메라 텐서에서 3x4 w2c 블록을 추출해 c2w로 inverse. `check_teleport_camera(extrinsics)`는 dl3dv 버전을 그대로 미러링 (non-finite, `|diff|>10` per-axis frame-to-frame jump, `||diff||>15` magnitude jump, `|t|>50` absolute translation 중 하나라도 걸리면 True). `build_re10k_meta`의 `scan(chunk_path)`이 length-mismatch 검사 통과 후 이 두 함수를 호출해 broken-camera scene을 skip하고, 결과 print에 `(skipped K length-mismatch / M bad-camera)` breakdown 표시. **기존 meta.csv는 재빌드 안 됨** — 새 필터를 적용하려면 해당 meta.csv를 지운 뒤 next dataset init 때 자동 rebuild.
- `src/model/diffusion_wrapper.py`에 `DiffusionWrapper._diagnose_batch_for_blacklist(batch, target_latents)` static method 추가. NaN loss 발생 시 batch의 context/target extrinsics·intrinsics·raw input(`latent` key)과 인코딩된 `target_latents`를 sample 단위로 검사해 anomaly 항목(reason, detail) list를 반환. 임계: `|t| > 50` (extrinsics translation, `check_teleport_camera`와 동일), raw input `|x| > 5`, encoded latent `|z| > 200`, 모든 텐서 non-finite check. `(scene, reason)` 단위로 dedup.

### Changed
- `src/model/diffusion_wrapper.py:training_step` NaN loss handler: 기존 "3회 누적 → `breakpoint()`" 동작을 제거하고, **NaN 발생할 때마다** `_diagnose_batch_for_blacklist`로 진단 → 데이터-쪽 anomaly가 잡히면 `dataset_dl3dv.append_blacklist`로 `{cfg.root}/train/blacklist.csv`에 append (scene 단위 dedup). 데이터셋이 `dl3dv`가 아닌 경우엔 진단만 로그하고 blacklist는 skip. 그래디언트 NaN guard(line ~1144)가 이미 optimizer step을 skip하므로 학습은 그대로 continue — auto-mode 친화적. `nan_loss_count`는 lifetime counter로 유지 (log 용).
- `CLAUDE.md` Commands → A. SceneTok 목록을 현재 레포의 실제 shell 파일명·exp_name으로 갱신. `train_scenetok_256.sh`/`train_scenetok_re10k.sh`/`train_scenetok_wan.sh`는 더 이상 존재하지 않으며 `train_scenetok_{dl3dv_256, dl3dv_wan_256, dl3dv_wan_480, re10k_256, re10k_vavdc, chunked}.sh`로 분기된 상태를 반영. `train_scenetok.sh` 설명도 잘못된 "DL3DV wan-wan scratch"에서 실제(`scenetok_va-vdc_re10k_scratch`)로 정정.
- `src/model/diffusion_wrapper.py:training_step`: **loss NaN/Inf counter + 3회 누적 시 `breakpoint()`** 추가. `self.nan_loss_count`를 `__init__`에서 0으로 초기화 (process-lifetime 누적), `loss.mean()` 직후에 `isnan/isinf` check → 발생 시 카운터++ + step·batch_idx·scenes를 stdout으로 출력, 3회째 발생 시 pdb 진입해 model state·batch·scene을 인터랙티브 inspect 가능. 기존 gradient NaN handler(`line 1144`)는 gradient만 skip하지 loss 자체의 발산 빈도/원인을 알 길이 없었던 gap을 메움. `continue`로 빠져나오면 학습 그대로 진행 (해당 step은 gradient guard가 자동 skip).
- `src/model/diffusion_wrapper.py:validation_step`: `self.generated`/`self.predicted`에 sample append 직전에 **NaN/Inf finite-mask guard + 진단 print** 추가. NaN/Inf가 있으면 count·min·max 출력 후 해당 sample들을 batch에서 drop, 전체가 NaN이면 batch 통째로 skip (return None). 학습 측 `Skipping Nans!` (line 1144)는 gradient만 skip하지 forward output을 sanitize하지 않아서 validation에 NaN이 그대로 흘러들어가 FVD `linalg.sqrtm`이 NaN-poisoned cov 위에서 Schur 분해 무한 spin으로 hang하던 진짜 root cause 대응. (`NaN + ε == NaN`이라 이전 ε regularization patch는 NaN-poisoned case에서 무력했음.)
- `src/model/metrics.py:update_fid/update_fvd`: 진입 시점에 finite-mask 필터 추가 (defense in depth). non-finite sample 발견 시 drop 개수 출력 후 제외, 전체 drop이면 silent skip. 위 source-side guard와 별개로 미래 다른 호출 경로(test_step, predict_step 등)에서 NaN이 들어와도 안전.
- `submodules/fvd/frechet_video_distance.py`: `calculate_fvd_from_activations`의 `linalg.sqrtm` 전 ε·I regularization 재부활 (이전 revert에서 빠졌던 패치). default ε `1e-10 → 1e-6`로 강화하고 무조건적으로 한 번 적용 후 sqrtm 호출. NaN/Inf fallback은 ε=1e-3로 재시도. 32 sample × 400-dim I3D feature → rank-deficient cov → scipy sqrtm의 Schur 분해 무한 spin (NaN이 아니라 진짜 hang)으로 train_scenetok_re10k_256 validation이 멈추던 문제 대응. FVD 절대값 변화는 ~1e-3 (negligible, sample-size << feature-dim에서 오는 estimator noise보다 작음). 모델 간 ranking은 보존.
- `src/dataset/dataset_re10k.py`: `__getitem__`의 `view_sampler.sample(...)` 호출을 try/except로 감싸 `ValueError`를 `return None`으로 graceful skip. 위 unbounded sanity-check 추가, scene별 frame 부족, OOB 인덱스 등 sampler 실패 케이스를 dataset 단에서 흡수해 worker crash 방지. `safe_collate` + step methods의 None guard와 결합해 None-safe path가 완성됨.
- `src/model/diffusion_wrapper.py`, `src/model/t2v_wrapper.py`: 모든 step 메서드(`training_step`, `validation_step`, `test_step`) 최상단에 `if batch is None: return None` guard 추가. `safe_collate`가 batch 전체 None 시 `None`을 반환하는 새 동작에 맞춰 — Lightning automatic optimization은 None을 반환하면 backward/optimizer.step()을 skip, validation_step은 이미 코드베이스에서 None 반환을 정상 패턴으로 쓰고 있어서 안전. DDP 동기화는 현재 모든 학습 스크립트가 `trainer.devices=1`이라 영향 없음. (T2VWrapper는 DiffusionWrapper subclass라 validation_step은 부모 guard 상속.)
- `src/dataset/data_module.py`: `safe_collate`에 `batch = [b for b in batch if b is not None]` 사전 필터 복원 + 전체가 None이면 `None` 반환 (caller가 skip 가능). val/test/predict dataloader에도 `collate_fn=safe_collate` 추가. 이유: `dataset_re10k.py.__getitem__`이 bad baseline / 잘못된 shape에서 `None`을 반환하는 경로(line 156, 174)는 그대로 살아있는데, 현재 `safe_collate`가 None을 사전 처리하지 않아 `default_collate`가 `'NoneType' object is not subscriptable`로 crash했음. 이전 revert로 train만 `safe_collate`를 쓰던 상태였는데, re10k가 val/test에서도 같은 `__getitem__`을 거치므로 양쪽 모두 None-safe로 통일.
- `config/experiment/custom/scenetok_va-wan_shift8_dl3dv_scratch_wide.yaml`: context/target shape을 256×448로 통일. `dataset.context_shape` `[256, 256] → [256, 448]`, `dataset.target_shape` `[480, 832] → [256, 448]`. 그에 맞춰 derived shapes도 조정: `model.denoiser.input_shape` `[30, 52] → [16, 28]`, `model.denoiser.camera.input_shape` `[240, 416] → [128, 224]`, `model.compressor.input_shape` `[16, 16] → [16, 28]`, `model.compressor.camera.input_shape` `[128, 128] → [128, 224]`. wandb tag도 `480x832 → 256x448`.

### Added
- `scripts/probe_lpips_fp16.py` — LPIPS metric이 학습 중 같은 값으로 수렴하는 증상 진단용 probe. (1) noise scale 0.001–0.05에서 fp32/fp16 LPIPS 비교 — fp16이 정상으로 monotonic하게 변하고 fp32와 1e-4 이내 일치하는 것 확인. (2) lin weight nonzero ratio 출력 — fp16 캐스트 후에도 underflow 없음(`14716138/14716160 ≈ 1.0`). (3) `tmp/wan_vae_recon/`의 real Wan VAE recon 10쌍에 대해 `Metric()`을 wrapper-동일 setup(`freeze` + `convert_to_buffer`)으로 instance화해서 bf16 입력으로 `compute_lpips` 호출 — per-scene `0.0123~0.0447`로 정상 변화, aggregate `0.024292`가 fp32 평균(`0.024125`)과 0.0002 이내 일치. 결론: **compute_lpips 함수 자체는 정상**. 남은 가설은 LightningModule의 precision-cast가 LPIPS buffer를 다운캐스트하는지 (H6).
- `scripts/dump_va_vae_recon.py` — VA-VAE (`checkpoints/vavae-imagenet256-f16d32-dinov2.pt`, f=16, latent_channels=32) 버전의 recon dump. 동일한 10개 re10k test scene을 **세 해상도**(256×256, 256×448, 480×832)에서 encode→decode 후 `tmp/va_vae_recon/`에 저장. 입력은 `(B,C,H,W)`, `(x*2-1)`로 [-1,1] 정규화 후 `.latent_dist.mode()`로 deterministic latent 사용. 결과: 256×256은 PSNR **31.40 dB** / L1 0.0176, 256×448은 33.67 dB / 0.0141, 480×832는 37.70 dB / 0.0088 — PSNR이 latent 토큰 수(256→448→1560)와 monotonically 비례. **VA-VAE의 학습 native 256×256에서도 recon이 가장 나쁨** (학습 해상도 근접성보다 f=16 bottleneck의 spatial 토큰 수가 dominant). Wan VAE도 같은 방향 (256×448→480×832에서 PSNR 6.4 dB 갭). VA-VAE는 raw latent magnitude가 큼 (z_std≈3.5, |z|max 최대 30) → 학습 경로의 `latent_mean`/`latent_std` 정규화 필요. |z|max는 큰 해상도에서 오히려 더 큼 → 작은 해상도 outlier 가설은 양 VAE에서 모두 지지되지 않음.
- `scripts/dump_wan_vae_recon.py` — re10k test 10 scene을 두 해상도(256×448, 480×832)에서 Wan2.2 VAE로 encode→decode 후 `tmp/wan_vae_recon/`에 `scene{NN}_{key}_{HxW}_{orig,recon}.png`로 저장. 학습 경로와 동일하게 `inputs * 2 - 1`로 [-1,1] 정규화 후 인코딩. scene별 PSNR/L1/z_std/|z|max와 해상도별 aggregate를 stdout으로 출력. 결과: 256×448은 PSNR 40.70 dB / L1 0.0062, 480×832는 47.07 dB / 0.0029로 **작은 해상도에서 recon 충실도가 일관되게 낮음** (latent z 통계는 두 해상도 거의 동일).
- `scripts/check_wan_vae_outliers.py` — Wan2.2 VAE에 re10k test scene N개를 통과시켜 latent의 NaN/Inf, `|mean|`, std, max|z|, kurtosis를 검사하고 per-channel std 분포까지 출력하는 진단 스크립트. `sys.path`를 자동으로 repo root에 추가하므로 어느 cwd에서 실행해도 동작 (단 데이터/ckpt 경로는 repo root 기준이라 거기서 실행 권장). 사용: `python scripts/check_wan_vae_outliers.py [num_scenes] [h,w]`.

### Changed
- `src/model/diffusion_wrapper.py:on_validation_end` — `self.metric(...)` 호출 직전에 LPIPS 내부 weight/buffer dtype과 pred/gt dtype을 출력하는 **임시 debug print** 추가. LPIPS metric이 학습 중 다른 모델들 간에 같은 값(예: 0.0244)으로 수렴하는 증상의 root cause를 H6(Lightning precision-cast가 LPIPS fp16 buffer를 bf16/fp32로 다운캐스트해서 VGG feature가 평탄해짐) vs 구조적 saturation으로 가려내기 위함. 진단 끝나면 제거 예정. 출력 예: `[LPIPS dtype] params=set() buffers={torch.float16} pred=torch.bfloat16 gt=torch.bfloat16`.
- `CLAUDE.md` 업데이트 — `/init` 결과. Commands 섹션을 SceneTok 학습 (`train_scenetok*.sh`, 현재 active line) / TI2V denoiser 학습의 두 그룹으로 재구성. Data 섹션에 re10k `meta.csv` 빌더, `{standard,unseen}` evaluation index split 자동 선택 (`(chunk_targets, val_seen)` 키), unbounded sampler `offset`/`chunk_targets` 동작 추가. 새 "Known gotchas" 서브섹션 — Wan2.2 VAE + 1024 scene_tokens scratch 학습의 `kl_weights` anchor 단계적 상향 패턴, FVD `sqrtm` hang 회피 노트.
- `src/dataset/dataset_re10k.py`를 **원본 대비 4가지 핵심 호환성 수정 + meta.csv build/load**만 남도록 최소화. ### Reverted
- 보조 안전망 패치들을 원본대로 되돌림 (사용자 요청):
  - `submodules/fvd/frechet_video_distance.py`: `calculate_fvd_from_activations`의 `linalg.sqrtm` 전 `1e-6·I` regularization 삭제 → 원래 단순 `sqrtm(f_cov.dot(s_cov))` 호출 + `isfinite` fallback만 유지.
  - `src/dataset/data_module.py`: `safe_collate`의 `batch = [b for b in batch if b is not None]` 사전 필터 제거. val/test/predict dataloader의 `collate_fn=safe_collate` 인자도 제거 (원래 train만 safe_collate 사용).
  - `src/dataset/dataset_re10k.py` `__getitem__`: view_sampler가 `ValueError` 던질 때 `return None`으로 감싸던 try/except 제거 → 다시 raise. `indices.max() >= len(example["images"])` 사전 검사도 제거.

### Changed
- `submodules/fvd/frechet_video_distance.py`: `calculate_fvd_from_activations`에서 `linalg.sqrtm` 호출 전에 cov에 `1e-6·I` regularization을 항상 적용. 32 sample × 수백 차원 I3D feature → rank-deficient cov → scipy `sqrtm`의 Schur 분해가 무한 스핀하던 validation hang을 회피. NaN/Inf 나오면 ε을 1e-3로 한 번 더 키워 retry.
- `config/experiment/custom/scenetok_wan-wan_lognorm_re10k_scratch.yaml`: re10k 256×448 wan-wan scratch가 step ~2300 부근에서 `compressor.scene_tokens` gradient NaN으로 발산하던 문제 대응. compressor에서 `init_large: false` + `init_small: true` (scene_tokens 초기 std 5.0 → 0.02). published va-vdc re10k는 num_scene_tokens=512인데 우리는 1024로 두 배라 같은 1e-10 KL anchor로는 부족.
